# AI Employee Dashboard - Platinum Tier

A comprehensive web-based dashboard for monitoring and managing your Autonomous AI Employee system.

## Features

### 📊 Dashboard Overview
- **Real-time Stats**: View pending tasks, approvals, emails, and completed items at a glance
- **System Status**: Monitor online/offline status of your AI Employee
- **Quick Access**: Navigate to all major sections from the sidebar

### 📋 Task Management
- View all pending tasks in Needs_Action folder
- Filter by type and priority
- Search tasks by name or content
- View detailed task information with metadata
- Mark tasks as complete with summary

### ✅ Approval Management
- View all pending approvals (email, social media, payments)
- Approve or reject actions with human-in-the-loop workflow
- Detailed approval previews before making decisions
- Statistics dashboard for approval types

### 📧 Email Triage
- View email drafts generated by Cloud Agent
- See sender, subject, and priority information
- Preview AI-generated draft replies
- Archive and forward functionality

### 📱 Social Media Management
- View posts for all platforms (LinkedIn, Facebook, Twitter, Instagram)
- Filter by platform
- See draft and published posts
- Post status tracking

### 💰 Accounting & Financial Reports
- Invoice tracking and management
- Transaction history
- Financial reports from Odoo ERP
- Quick financial summary

### 🏥 System Health Monitor
- Real-time CPU, memory, and disk usage
- Process health checks
- Auto-restart statistics
- System logs viewer
- Uptime tracking

### 📄 CEO Briefings
- View generated CEO briefings
- Weekly business audit reports
- Download briefing documents
- Historical briefing archive

### ⚙️ Settings
- Dashboard configuration
- Auto-refresh interval settings
- Light/Dark theme toggle
- Vault sync operations
- Cache management

## Installation

### Prerequisites
- Python 3.13+
- pip package manager

### Setup

1. **Install dependencies:**
```bash
cd Hack-03-AI-Employee-giaic
pip install -r requirements.txt
```

2. **Ensure your AI Employee vault exists:**
```bash
# The dashboard expects the AI_Employee_Vault folder to exist
# with the standard folder structure:
# - Inbox/
# - Needs_Action/
# - Pending_Approval/
# - Done/
# - Updates/
# - Signals/
# - Dashboard.md
```

3. **Start the dashboard:**
```bash
python dashboard/app.py
```

4. **Access the dashboard:**
Open your browser and navigate to:
```
http://localhost:5000
```

## Usage

### Running in Production

For production deployment, use gunicorn:

```bash
gunicorn --bind 0.0.0.0:5000 --workers 2 dashboard.app:app
```

### Running as a Background Service

**Linux (systemd):**

Create `/etc/systemd/system/ai-employee-dashboard.service`:

```ini
[Unit]
Description=AI Employee Dashboard
After=network.target

[Service]
User=your-user
WorkingDirectory=/path/to/Hack-03-AI-Employee-giaic
ExecStart=/path/to/venv/bin/gunicorn --bind 0.0.0.0:5000 dashboard.app:app
Restart=always

[Install]
WantedBy=multi-user.target
```

Then:
```bash
sudo systemctl enable ai-employee-dashboard
sudo systemctl start ai-employee-dashboard
```

**Windows (Task Scheduler):**

Create a batch file `start_dashboard.bat`:
```batch
@echo off
cd /d "%~dp0"
call venv\Scripts\activate
python dashboard\app.py
```

Schedule it to run at startup using Task Scheduler.

## API Reference

The dashboard provides a REST API for programmatic access:

### Vault Operations
- `GET /api/vault/status` - Get overall vault status
- `GET /api/tasks/list` - List all pending tasks
- `GET /api/tasks/<name>` - Get task details
- `POST /api/tasks/<name>/complete` - Mark task as complete

### Approvals
- `GET /api/approvals/list` - List pending approvals
- `POST /api/approvals/<name>/approve` - Approve an action
- `POST /api/approvals/<name>/reject` - Reject an action

### Email
- `GET /api/email/list` - List email triage items

### Social Media
- `GET /api/social/list` - List social media posts

### Accounting
- `GET /api/accounting/summary` - Get accounting summary

### Health
- `GET /api/health/status` - Get system health status
- `GET /api/health/logs` - Get recent logs

### Briefings
- `GET /api/briefings/list` - List CEO briefings
- `GET /api/briefings/<name>` - Get briefing content

### Signals
- `GET /api/signals/list` - List urgent signals/alerts

## Integration with AI Employee

The dashboard integrates with your AI Employee system through:

1. **Vault Skills**: Uses `skills/vault_skills.py` for vault operations
2. **Log Manager**: Uses `log_manager.py` for centralized logging
3. **File System**: Direct access to vault folders for real-time data

## Security Considerations

- The dashboard runs locally by default
- For remote access, use HTTPS and authentication
- Never expose the dashboard to the public internet without proper security
- API endpoints should be protected in production

## Troubleshooting

### Dashboard won't start
- Check that all dependencies are installed: `pip install -r requirements.txt`
- Ensure the vault folder exists
- Check logs in `logs/dashboard.log`

### Data not loading
- Verify the vault folder structure is correct
- Check that the AI Employee orchestrator is running
- Review logs for errors

### Slow performance
- Reduce auto-refresh interval in settings
- Check system resources (CPU, memory, disk)
- Review health monitor logs
